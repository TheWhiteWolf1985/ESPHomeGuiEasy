4  Gestione dei Progetti

4.1  Creazione di un nuovo progetto

  1. Menu File → Nuovo progetto.
  2. Scegliere la directory che conterrà la cartella progetto.
  3. Digitare il nome progetto; verrà creata una cartella con lo stesso nome.
  4. All’interno viene copiato il template.yaml di base.
  5. Il compilatore viene configurato automaticamente per puntare alla nuova cartella.

4.2  Apertura di un progetto esistente

Menu File → Apri progetto… consente di selezionare direttamente il file .yaml del progetto. ESPHomeGuiEasy:

  • Carica il contenuto YAML nell’editor.
  • Sincronizza le schede Impostazioni, Moduli e Sensori.
  • Memorizza la cartella come directory di compilazione.
La funzione “Apri progetto” accetta anche file con estensione .yml.

4.3  Importa/Esporta progetto (ZIP)

OperazioneDescrizione
Importa progetto…Seleziona un archivio .zip; l’app lo estrae in una cartella e carica automaticamente il file YAML.
Esporta progetto…Comprimi l’intera cartella progetto in un nomeprogetto.zip, escludendo la cartella .pioenvs.
[12:11:07] INFO Esportazione in corso...
[12:11:12] SUCCESS Progetto esportato come sensor_studio.zip (1,2 MB)
Best practice: esporta sempre un progetto prima di condividerlo, così includerai anche eventuali asset aggiuntivi (icone, immagini, script Lua, ecc.).

4.4  Struttura delle cartelle di progetto

La struttura generata da ESPHomeGuiEasy segue questa convenzione:

my_project/
├── my_project.yaml
├── assets/           # immagini e file statici aggiuntivi
├── src/              # codice C++ opzionale
└── .pioenvs/         # cartella build creata da ESPHome (ignorata in ZIP)
Mantieni separati file generati (.pioenvs) e sorgenti per rendere l’esportazione più snella.
